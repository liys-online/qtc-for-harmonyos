# 详细设计：路线图与风险

---

## 1. 能力优先级（建议）

| 优先级 | 能力 | 说明 |
|--------|------|------|
| **P0** | SDK/Node/hdc 路径检测 + 设置页 | 与 Android SDK 页同等地位 |
| **P0** | Kit + Qt + Toolchain 可组成可构建 Kit | 阻塞所有后续 |
| **P0** | 真机枚举 + HAP 构建 + hdc 安装 | 最小闭环 |
| **P1** | 构建/部署日志与错误码友好化 | 提升可维护性 |
| **P1** | Run：一键启动应用 | 不必含调试 |
| **P2** | 模拟器 / 多设备并发安装 | 视官方工具链能力 |
| **P2** | 调试器深度集成（C++/ArkTS） | 工作量大，依赖上游 |
| **P2** | 向导对齐 DevEco 模板升级 | 持续跟进 |

---

## 2. 已知缺口（随实现勾选）

- [ ] 与 Android **同等粒度** 的 Run/Debug 体验  
- [ ] 模拟器生命周期管理（若不做，需在文档标明「使用外部模拟器」）  
- [ ] 应用签名与发布渠道（对齐 Android keystore 管理可选实现）  
- [ ] 自动化测试覆盖 hdc/hvigor 版本矩阵  

---

## 3. 风险与假设

| 风险 | 缓解 |
|------|------|
| 官方 CLI 变更 | 版本检测 + 发行说明；可选「已知兼容版本」提示 |
| Qt Creator API 再变 | `VERSIONING.md` + `compat/` 追加源文件 |
| macOS / Windows 路径与权限差异 | 分平台 CMake（已部分用于 libusb） |
| hdc/USB 稳定性 | 重试与明确错误信息；文档说明驱动/权限 |

---

## 4. 与 Android 插件协作（产品层）

- 用户可在同一 IDE 中同时安装 Android 与 Harmony 插件；**设置页分区** 应清晰，避免键名冲突（Harmony 使用独立 settings key 前缀，见 `harmonyconstants`）。

---

## 5. 文档维护

- 每完成一个 P0/P1 项，更新 `DESIGN-DETAIL-ANDROID-MAPPING.md` 中对应 **状态** 列。  
- 大版本升级 Qt Creator 时，同步检查 **扩展点签名** 与本文档「任务模型」小节。
